# 구글시트 연동 설정 가이드 (4단계)

체크리스트 체크 여부 / 예약 완료 상태 / 현지 메모·가계부를 가족 3인이
**같은 데이터로 공유**하도록 구글시트를 "가벼운 백엔드"로 사용합니다.
아래 순서대로 딱 한 번만 설정하면 됩니다. (소요 시간 약 5분)

## 1. 구글 스프레드시트 만들기
1. [sheets.google.com](https://sheets.google.com) 에서 새 스프레드시트 생성
2. 이름은 자유롭게 (예: `유럽여행2026_데이터`)
3. 시트(탭)는 미리 만들 필요 없습니다. 아래 Apps Script가 처음 실행될 때
   `Checklist`, `Reservations`, `Memos` 3개 탭을 자동으로 만들어줍니다.

## 2. Apps Script 코드 붙여넣기
1. 방금 만든 스프레드시트에서 상단 메뉴 **확장 프로그램 → Apps Script** 클릭
2. 기본으로 열린 `Code.gs` 파일의 내용을 전부 지우고,
   이 폴더에 있는 **`Code.gs` 파일 내용을 그대로 복사해서 붙여넣기**
3. 상단의 저장 아이콘(💾) 클릭 (프로젝트 이름은 아무거나 지정)

## 3. 웹앱으로 배포하기
1. Apps Script 화면 오른쪽 위 **배포 → 새 배포** 클릭
2. 유형 선택(⚙️ 톱니바퀴)에서 **웹앱** 선택
3. 설정값:
   - **설명**: 아무거나 (예: `유럽여행 웹앱 v1`)
   - **다음 사용자로 실행**: `나(내 계정)`
   - **액세스 권한이 있는 사용자**: `모든 사용자` (Anyone)
4. **배포** 클릭 → 구글 계정 권한 승인 창이 뜨면 허용
   (Google이 "확인되지 않은 앱" 경고를 띄울 수 있는데, `고급` → `안전하지 않은 페이지로 이동` 선택하면 됩니다.
   본인이 만든 개인용 스크립트라 발생하는 안내이며 문제 없습니다.)
5. 배포가 끝나면 **웹앱 URL**이 표시됩니다.
   `https://script.google.com/macros/s/AKfycb.../exec` 형태의 URL을 복사해두세요.

> 이후 Code.gs를 수정하게 되면 **배포 → 배포 관리 → 수정(연필 아이콘) → 새 버전** 으로
> 다시 배포해야 변경사항이 반영됩니다. (URL은 그대로 유지됩니다.)

## 4. 사이트에 URL 등록하기
1. 배포된 사이트(GitHub Pages)에 접속
2. 오른쪽 위 톱니바퀴(설정) 아이콘 클릭
3. **Apps Script 웹앱 URL** 입력창에 3단계에서 복사한 URL을 붙여넣기
4. **저장하고 지금 동기화** 클릭 → "동기화 완료" 메시지가 뜨면 성공

이제 이 URL을 가족 3인 모두가 각자의 폰에서 **같은 값으로** 설정하면,
체크리스트 체크, 예약 완료 표시, 메모/지출 기록이 모두 같은 구글시트에 쌓이고 공유됩니다.

## 동작 방식 (참고)
- 체크리스트를 체크하거나 예약을 완료로 표시하면 → 즉시 이 기기에 저장(오프라인에서도 동작) + 동시에 구글시트로 전송
- 앱을 열 때마다 구글시트에서 최신 데이터를 먼저 가져와 덮어쓰기 때문에, 다른 가족이 미리 체크한 내용도 곧바로 보입니다
- 인터넷이 안 되는 상황에서 체크해도 이 기기에는 정상 저장되며, 나중에 인터넷이 연결된 후 설정 화면에서
  "지금 동기화"를 눌러주면 다시 최신 상태로 맞출 수 있습니다 (현재 버전은 자동 재전송까지는 지원하지 않습니다)

## 문제 해결
| 증상 | 확인할 것 |
|---|---|
| "동기화에 실패했습니다" | URL을 다시 확인 (`.../exec`로 끝나야 함), 배포 시 액세스 권한이 "모든 사용자"인지 확인 |
| 체크했는데 다른 가족 폰에는 안 보임 | 모든 기기가 **완전히 동일한 URL**을 설정에 입력했는지 확인, 각자 설정 화면에서 "지금 동기화" 눌러보기 |
| 구글시트에 값이 이상하게 들어감 | Code.gs를 그대로(수정 없이) 붙여넣었는지, 새 버전으로 재배포했는지 확인 |
